# GamePoch Content Copilot

面向游戏发行团队、内容运营、视频剪辑、活动跟拍和海外资讯整理岗位的内容生产工作台。

目标不是做通用聊天机器人，而是把日常内容工作流压缩成一条可执行链路：

```text
发现热点 -> 分析价值 -> 生成标题/脚本 -> 规划活动内容 -> 导出作战简报
```

## 核心价值

- 自动聚合 Steam 热点、Steam 新品/折扣、IGN、GameSpot、PC Gamer 等海外资讯。
- 将原始数据转成“今天值得做什么内容”的运营判断。
- 一键生成 B站、抖音、小红书、公众号标题。
- 一键生成 15 秒、30 秒、60 秒短视频脚本。
- 支持输入任意游戏名，优先匹配精选内容或 Steam 搜索结果，再生成内容包。
- 生成封面文案、采访提纲，以及 Markdown / Word 内容作战简报。
- 新增游戏活动内容准备助手，覆盖活动前、活动当天、活动后三个阶段。
- 新增7日内容发布排期，把热点和活动准备转成平台任务、负责人和素材清单。
- 对 Steam 实时榜做演示安全过滤，避免面试 Demo 混入不适合展示的内容。

## 当前模块

### 1. 今日工作台

展示当天内容机会数量、推荐短视频选题、海外资讯数量和高优先级风险。

### 2. 热点发现

从 Steam 和海外资讯中筛选适合当天跟进的内容机会，并给出：

- 为什么值得做
- 推荐切入角度
- 适合产出形式

### 3. 内容生成

对选中的热点自动生成：

- B站标题
- 抖音标题
- 小红书标题
- 公众号标题
- 15/30/60 秒脚本
- 封面文案
- 采访问题

也支持手动输入游戏名或资讯主题生成内容包。

如果输入的游戏能在精选池或 Steam 中匹配到，系统会自动带入更完整的游戏信息和评价概况；如果没有匹配到，也会降级为普通主题生成，保证现场演示不断线。

### 4. 游戏活动内容准备助手

用高质量 Demo 数据展示游戏行业活动规划能力，覆盖：

- Steam Next Fest
- Gamescom
- The Game Awards
- Summer Game Fest
- PlayStation State of Play
- Nintendo Direct

每个活动包含：

- 活动时间、活动类型、适合关注的游戏品类
- 内容机会和风险提示
- 活动前：预热视频选题、平台标题、海外资讯摘要、采访问题、物料清单
- 活动当天：必拍镜头、采访对象、B-roll 素材建议、现场短视频切片建议
- 活动后：30 秒短视频脚本、活动复盘日报、公众号/小红书文案、团队内部汇报摘要

### 5. 7日内容发布排期

自动生成一周内容执行计划，包含：

- 日期和发布时段
- 平台：B站、抖音、小红书、公众号、团队内部
- 内容形式：短视频、热点解析、图文笔记、日报条目、活动准备
- 负责人角色：视频剪辑、内容运营、英语资讯、活动跟拍
- 素材需求和执行目标

排期可复制或导出 Markdown，适合团队分工和每日跟进。

### 6. 内容作战简报

自动生成 Markdown / Word 简报，可复制或导出，用于团队晨会、日报、周报或 PPT 素材整理。

简报会合并：

- 今日热点优先级
- 推荐执行动作
- 活动内容准备
- 风险提醒
- 可直接复制给团队的任务分工

Word 版简报适合发给负责人、合作伙伴或作为正式汇报材料底稿。

## 启动方式

```bash
cd E:\gamepoch-tools
pip install -r requirements.txt
python app.py
```

浏览器打开：

```text
http://localhost:8000
```

也可以双击 `start.bat` 启动。

## 演示前自检

启动服务后，另开一个 PowerShell 窗口运行：

```powershell
cd E:\gamepoch-tools
.\CHECK_DEMO.ps1
```

脚本会检查首页、热点发现、自定义内容生成、活动规划、发布排期、Markdown 简报和 Word 简报。

也可以直接访问：

```text
http://localhost:8000/api/demo/status
```

如果返回 `ready: true`，说明 Demo 核心链路可以演示。

## 面试后发送材料

- `PACKAGE_FOR_SEND.md`：发给对方前的材料整理、演示顺序和注意事项。
- `SEND_MESSAGE_TEMPLATE.md`：微信/邮件简短话术。
- `DEMO_GUIDE.md`：3分钟演示路线。
- `POST_INTERVIEW_NOTE.md`：作品定位说明。

## 技术栈

- 后端：FastAPI + httpx
- 前端：Tailwind CSS + Chart.js + Lucide icons
- 数据源：Steam Store API、Steam Reviews、IGN RSS、GameSpot RSS、PC Gamer RSS
- 报告导出：Markdown、Word，保留原游戏舆情 Word 报告接口能力

## Demo 定位

这个 Demo 面向小型游戏发行团队，重点体现：

```text
原本需要人工花几小时整理的热点、标题、脚本、活动清单、发布排期和汇报简报，现在可以在一个工作台里快速完成。
```
