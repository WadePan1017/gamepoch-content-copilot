# GamePoch Content Copilot 演示指南

## 推荐演示顺序

1. 打开首页
   - 说明这是基于面试后岗位理解做的非正式 Demo。
   - 强调目标是帮助内容团队节省热点整理、选题、脚本和日报时间。

2. 今日工作台
   - 看今日内容机会、短视频选题、海外资讯和风险提醒。
   - 说明老板/负责人可以一眼看到今天该做什么。

3. 热点发现
   - 点击“影之刃零”或“艾尔登法环”。
   - 展示为什么值得做、推荐切入角度和适合产出。

4. 内容生成
   - 输入 `Elden Ring` 或 `Split Fiction`。
   - 展示 B站/抖音/小红书标题、30 秒脚本、封面文案和采访问题。

5. 活动规划
   - 选择 Gamescom。
   - 展示活动前准备、当天拍摄清单和活动后产出。

6. 发布排期
   - 展示 7 日内容发布计划。
   - 强调它把内容生成结果变成团队可执行任务。

7. 简报导出
   - 展示内容作战简报。
   - 说明可导出 Markdown / Word，用作日报、周报或负责人汇报。

8. 演示话术
   - 展示这是为了让对方快速理解作品价值。

## 30秒讲法

这个 Demo 是我基于面试后对星游纪/GamePoch 游戏发行内容岗位的理解做的非正式作品。它不是聊天机器人，而是一个内容作战室：把热点发现、内容选题、短视频脚本、活动拍摄准备、发布排期和简报整理放到一个工作台里，目标是帮助小团队节省内容生产时间。

## 重点不要讲太多技术

少讲：

- FastAPI
- Tailwind
- API 细节
- 代码实现

多讲：

- 岗位理解
- 内容效率
- 活动跟拍准备
- 海外资讯整理
- 如何帮团队节省时间
