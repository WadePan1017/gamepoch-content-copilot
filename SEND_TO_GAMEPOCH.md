# 发给星游纪 / 浸游的补充说明

## 推荐发送话术

陈老师您好，我是之前面试视频剪辑岗位的候选人。

面试后我复盘了一下，感觉贵司实际需要的不只是单纯剪辑，而是能帮助游戏发行团队更快完成"发现热点、判断选题、生成脚本、整理素材"的内容生产能力。

所以我做了一个小型 Demo：GamePoch Content Copilot。

它核心解决一个问题：**从 Steam 热点到可发布的 30 秒脚本，只需要点 3 下。**

在线地址：
https://gamepoch-content-copilot.onrender.com

它主要演示：

- 自动抓取 Steam 热销榜、新品、折扣和海外游戏资讯
- 选一个热点，AI 自动生成 B站/抖音/小红书标题和 15s/30s/60s 短视频脚本
- 一键导出剪辑工作单和团队简报

这个 Demo 不是为了展示技术，而是想表达我对贵司内容发行岗位的理解：如果能把原本每天人工找热点、想标题、写脚本、做日报的流程压缩到一个工作台里，视频剪辑和内容运营的效率会高很多。

Render 免费服务首次打开可能需要几十秒冷启动，如果页面第一次加载较慢，稍等一下刷新即可。

谢谢您，也希望有机会继续沟通。

## 2 分钟演示路径

1. 打开首页，看"3步工作流"。
   - 重点说明：不是静态页，后端实时抓 Steam 和海外资讯。

2. 点"查看今日热点"，选一个游戏。
   - 推荐选热度最高的那个。
   - 重点说明：每个热点都有来源、建议动作、可产出。

3. 点"生成内容包"，看结果。
   - 重点说明：标题、脚本、素材清单都是 AI 实时生成的，每次不同。

4. 回首页输入一个游戏名。
   - 推荐输入：`拳皇15`、`艾尔登法环`、`Phantom Blade Zero`。

5. 点"导出简报"。
   - 重点说明：可以给负责人看，也能作为日报/周报草稿。

## 发送前检查

- 先打开一次 Render 地址，让免费服务从休眠中启动。
- 点击刷新，确认右上角显示"实时xx条 / 资讯xx条"。
- 点击热点生成一次内容包，确认显示"AI生成"标签。
- 不要只发 GitHub Pages 静态页，优先发 Render 地址。
