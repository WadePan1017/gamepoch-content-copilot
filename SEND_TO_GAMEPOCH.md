# 发给星游纪 / 浸游的补充说明

## 推荐发送话术

陈老师您好，我是之前面试视频剪辑岗位的候选人。

面试后我复盘了一下，感觉贵司实际需要的不只是单纯剪辑，而是能帮助游戏发行团队更快完成“发现热点、判断选题、生成脚本、整理海外资讯、准备活动内容和输出简报”的内容生产能力。

所以我做了一个小型 Demo：GamePoch Content Copilot。

在线地址：

https://gamepoch-content-copilot.onrender.com

它主要演示：

- 实时抓取 Steam 热点和海外游戏资讯
- 根据热点生成 B站 / 抖音 / 小红书标题
- 自动生成 15秒 / 30秒 / 60秒短视频脚本
- 输出剪辑工作单、素材清单、发布前风险核查
- 为 Steam Next Fest、Gamescom 等活动生成内容准备清单
- 自动整理团队简报和发布排期

这个 Demo 不是为了展示技术，而是想表达我对贵司内容发行岗位的理解：如果能把原本每天人工找热点、想标题、写脚本、整理素材和做日报的流程压缩到一个工作台里，视频剪辑和内容运营的效率会高很多。

Render 免费服务首次打开可能需要几十秒冷启动，如果页面第一次加载较慢，稍等一下刷新即可。

谢谢您，也希望有机会继续沟通。

## 2 分钟演示路径

1. 打开首页，先看右上角“实时数据”。
   - 重点说明：不是纯静态页，后端会抓 Steam 和海外 RSS。

2. 看“实时数据源”卡片。
   - 重点说明：Steam / IGN / GameSpot / PC Gamer / Kotaku。

3. 进入“热点发现”。
   - 重点说明：每个热点都有来源、建议动作、可产出。

4. 点击一个热点的“生成内容包”。
   - 重点说明：自动生成标题、脚本、素材清单、剪辑工作单。

5. 回首页输入一个游戏名。
   - 推荐输入：`拳皇15`、`艾尔登法环`、`Phantom Blade Zero`。

6. 打开“活动规划”。
   - 重点说明：活动前、活动当天、活动后都能生成内容准备。

7. 打开“简报导出”。
   - 重点说明：可以给负责人看，也能作为日报/周报草稿。

## 发送前检查

- 先打开一次 Render 地址，让免费服务从休眠中启动。
- 点击刷新，确认右上角显示“实时xx条 / 资讯xx条”。
- 点击热点生成一次内容包，确认内容正常。
- 不要只发 GitHub Pages 静态页，优先发 Render 地址。
