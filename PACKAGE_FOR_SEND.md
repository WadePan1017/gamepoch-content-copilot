# 面试后补充作品发送包

## 当前最推荐的发送方式

先不要一上来发一大堆源码。建议先发一段克制说明，再附一个可访问的演示链接、截图或压缩包。

推荐定位：

```text
这是我基于面试后对星游纪/GamePoch 游戏发行内容岗位的理解做的非正式 Demo，
不是公司内部系统，也不是正式产品，主要用于补充展示我对岗位工作的理解。
```

## 可以发给对方的内容

1. 一句话说明：
   - GamePoch Content Copilot 是一个面向游戏发行内容团队的效率 Demo。

2. 三个价值点：
   - 热点发现：聚合 Steam / 海外资讯，判断今天值得做什么内容。
   - 内容生成：生成 B站、抖音、小红书标题和短视频脚本。
   - 活动准备：为 Gamescom、Steam Next Fest 等活动生成拍摄清单、采访问题和复盘简报。

3. 展示材料：
   - 首页截图
   - 内容生成截图
   - 活动规划截图
   - 7日排期或 Word 简报

## 本地演示前自检

先启动服务：

```powershell
cd E:\gamepoch-tools
python app.py
```

然后另开一个 PowerShell 窗口运行：

```powershell
cd E:\gamepoch-tools
.\CHECK_DEMO.ps1
```

看到“自检通过：Demo 可以演示。”后，再打开：

```text
http://localhost:8000
```

## 3分钟演示顺序

1. 今日工作台：说明它不是聊天机器人，而是内容作战室。
2. 热点发现：点一个热点，展示为什么值得做。
3. 内容生成：输入 `Elden Ring`，展示平台标题和30秒脚本。
4. 活动规划：选择 Gamescom，展示活动前、当天、活动后清单。
5. 发布排期：展示7日任务怎么分给剪辑、运营、英语资讯和活动跟拍。
6. 简报导出：展示 Markdown / Word 简报。

## 不建议说的话

- 不要说“我已经给你们公司做了一套内部系统”。
- 不要说“这个可以直接替代你们员工”。
- 不要催结果。
- 不要过度强调技术栈。

## 建议强调的话

- 我面试后复盘了岗位，感觉它不只是剪辑，还涉及内容发现、活动准备、海外资讯整理和团队汇报。
- 这个 Demo 想展示我对游戏发行内容工作的理解。
- 如果有机会加入，我希望自己不只是完成剪辑任务，也能帮团队提高内容生产效率。
